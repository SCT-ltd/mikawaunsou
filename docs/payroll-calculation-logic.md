# 三川運送 給与計算ロジック説明書

作成日：2026年4月22日

---

## 1. 給与計算の全体フロー

```
基本給
  ＋ 各種手当（固定手当 + 月次計算手当）
  ＝ 総支給額
  
総支給額
  → 健康保険料（標準報酬月額ベース ※固定）
  → 厚生年金保険料（標準報酬月額ベース ※固定）
  → 雇用保険料（総支給 × 料率 ※毎月変動）
  ＝ 社会保険料控除後の金額

社会保険料控除後の金額
  → 所得税（月額表甲欄 ※毎月変動）
  → 住民税（毎月固定・年1回更新）
  → その他控除（積立金等）
  ＝ 差引支給額
```

## 2. 基本給の計算

### 日給制（三川運送の一般社員）

| 区分 | 単価 | 計算方法 |
|------|------|----------|
| 平日 | 9,808円/日 | 東京最低賃金1,163円 × 8時間（実際値） |
| 土曜出勤 | 12,260円/日 | 平日単価 × 1.25（休日割増） |
| 日祝出勤（時間） | 1,655円/時 | 時給換算 |

**計算例（22日出勤の場合）：**
```
基本給 = 22日 × 9,808円 = 215,776円
```

## 3. 手当の種類

### 固定手当（毎月同額・社員マスタに登録）

| 手当名 | 内容 | 例：岡田さん |
|--------|------|------------|
| 無事故手当 | 無事故継続ボーナス | 20,000円 |
| 愛車手当 | 自家用車使用補助 | 20,000円 |
| 交通費 | 通勤交通費 | 20,000円 |

> ⚠️ **注意**: 以下はBW計算式で自動計算されるため手当テーブルに登録してはいけない
> - 職務手当（固定残業代）→ BW式の `fixedOvertimeAmount` として別管理
> - 休日出勤手当 → 月次実績から自動計算
> - 深夜手当 → 月次実績から自動計算
> - 業績手当 → BW計算式から自動計算

### 月次計算手当（月次実績から計算）

| 手当名 | 計算方法 | 参照元 |
|--------|----------|--------|
| 残業手当（固定残業を超えた分） | (残業時間 - 固定残業時間) × 時間単価 | 月次実績 |
| 休日出勤手当 | 休日出勤日数 × 12,260円 | 月次実績 |
| 深夜手当 | 深夜時間 × (基本時給 × 0.25) | 月次実績 |

## 4. ブルーウィング（BW）歩合給計算

**対象社員：** 岡田一徳（EMP007）

### 計算ステップ

```
① A = BW売上 × 歩合率
   例: 827,000円 × 37.5% = 310,125円

② B = 基本給 + 固定残業代 + 休日出勤手当
   例: 215,776 + 50,000 + 24,520 = 290,296円

③ 業績手当 = 切り捨て1,000円単位(max(0, A - B))
   例: max(0, 310,125 - 290,296) = 19,829 → 20,000円

④ 実残業代 = (実残業時間 - 固定残業時間) × 時間単価
   例: (70.5h - 25h) × 2,111円 = 96,051円

⑤ 総支給 = 基本給 + 固定残業代 + 実残業代 + 深夜手当
           + 休日出勤手当 + 固定手当合計 + 業績手当
   例: 215,776 + 50,000 + 96,051 + 1,899 + 24,520 + 60,000 + 20,000
     = 468,246円  ✅（3月公式明細と一致）
```

### 検証済み数値（2026年3月 岡田さん）

| 項目 | 金額 | 備考 |
|------|------|------|
| 基本給 | 215,776円 | 22日 × 9,808円 |
| 固定残業代（職務手当） | 50,000円 | BW固定 |
| 実残業代 | 96,051円 | 45.5h × 2,111円 |
| 深夜手当 | 1,899円 | 月次実績から計算 |
| 休日出勤手当 | 24,520円 | 2日 × 12,260円 |
| 無事故手当 | 20,000円 | 固定 |
| 愛車手当 | 20,000円 | 固定 |
| 交通費 | 20,000円 | 固定 |
| 業績手当 | 20,000円 | BW計算式 |
| **総支給** | **468,246円** | ✅ |

## 5. 社会保険料の計算

### 5-1. 健康保険料

- **計算ベース**: 標準報酬月額（算定基礎届により年1回9月に更新、原則固定）
- **料率**: 東京支部 協会けんぽ
  - 健康保険本体: 9.85%（従業員負担 4.925%）
  - 介護保険（40〜64歳のみ）: 別途追加
- **計算式**: `round50sen(標準報酬月額 × 4.925%)`

> 📌 **重要**: 毎月の給与が増減しても、標準報酬月額が変わらない限り保険料は変わらない

| 岡田さんの場合 | 値 |
|--------------|-----|
| 標準報酬月額 | 410,000円（等級設定） |
| 健康保険料（本人負担） | 20,193円 |
| 介護保険料（40歳以上適用） | 3,321円 |
| 合計（healthInsuranceMonthly） | 23,514円 |

### 5-2. 厚生年金保険料

- **計算ベース**: 標準報酬月額（上限650,000円）
- **料率**: 18.30%（従業員負担 9.15%）
- **計算式**: `round50sen(min(標準報酬月額, 650,000) × 9.15%)`

| 岡田さんの場合 | 値 |
|--------------|-----|
| 標準報酬月額 | 410,000円 |
| 厚生年金保険料 | 37,515円 |

### 5-3. 雇用保険料

- **計算ベース**: **毎月の総支給額**（標準報酬月額ではない）
- **料率**: 0.55%（令和6年度〜）
- **計算式**: `round50sen(総支給額 × 0.55%)`
- **特徴**: 総支給が変わるたびに毎月変動する

| 月 | 総支給 | 雇用保険（0.55%） |
|----|--------|-----------------|
| 3月 | 468,246円 | 2,575円 |
| 4月 | 370,000円 | 2,035円 |

## 6. 所得税の計算

- **計算ベース**: 「総支給 − 社会保険料合計」後の課税対象額
- **方法**: 国税庁 令和7年分 給与所得の源泉徴収税額表（月額表・甲欄）
- **影響要素**: 扶養人数（配偶者含む）
- **特徴**: 総支給が変わるたびに変動する

```
課税対象額 = 総支給 − (健康保険 + 介護保険 + 厚生年金 + 雇用保険)
所得税    = 月額表甲欄で課税対象額と扶養数から参照
```

| 岡田さん3月の計算 | 金額 |
|----------------|------|
| 総支給 | 468,246円 |
| 社保合計 | 63,604円 |
| 課税対象額 | 404,642円 |
| 所得税（扶養0人） | 12,920円 |

## 7. 控除合計と差引支給額

```
控除合計 = 健康保険 + 厚生年金 + 雇用保険 + 所得税 + 住民税 + その他控除

差引支給額 = 総支給 − 控除合計
```

| 岡田さん3月 | 金額 |
|-----------|------|
| 社保合計 | 63,604円 |
| 所得税 | 12,920円 |
| 住民税 | 18,400円 |
| その他控除（積立金等） | 1,000円 |
| **控除合計** | **95,924円** |
| **差引支給額** | **372,322円** ✅ |

## 8. 「明細入力」タブの動作仕様

現在の仕様と推奨動作：

| 項目 | 現在の動作 | 理想の動作 |
|------|-----------|-----------|
| 健康保険料 | `healthInsuranceMonthly`手動値を優先 ❌ | 標準報酬月額×料率でリアルタイム計算 ✅ |
| 厚生年金料 | `pensionMonthly`手動値を優先 ❌ | 標準報酬月額×料率でリアルタイム計算 ✅ |
| 雇用保険料 | BW社員はsavedPayroll値を参照 ❌ | 総支給×0.55%でリアルタイム計算 ✅ |
| 所得税 | `incomeTaxMonthly`手動値を優先 ❌ | 課税対象額→税額表でリアルタイム計算 ✅ |

### 改善要件

1. **`standardRemuneration`（標準報酬月額）を各社員マスタに登録** する
   - 岡田さん: 410,000円（現在 `0` のまま → 要更新）
   
2. **手動設定フィールドを廃止**し、常にフォーミュラベースで計算する
   - `healthInsuranceMonthly` → 削除（`standardRemuneration × 料率`で代替）
   - `pensionMonthly` → 削除（同上）
   - `incomeTaxMonthly` → 削除（`afterInsuranceSalary → 税額表`で代替）

3. **雇用保険は常に** `grandTotal × 0.55%` で計算（savedPayroll参照をやめる）

## 9. データ不整合の注意事項

### BW社員の手当テーブル登録ルール

BW計算式で**自動計算される項目**は、手当テーブルに登録しない：

| 登録してはいけないもの | 理由 |
|--------------------|------|
| 職務手当（50,000円） | `bluewingFixedOvertimeAmount`としてBW式で管理 |
| 残業手当 | 月次実績から自動計算 |
| 休日出勤手当 | 月次実績から自動計算 |
| 深夜手当 | 月次実績から自動計算 |
| 業績手当 | BW計算式から自動計算 |

手当テーブルに登録するのは**固定カスタム手当のみ**：
- 無事故手当: 20,000円
- 愛車手当: 20,000円
- 交通費: 20,000円
