あなたは優秀なフロントエンドエンジニアです。
以下の要件と仕様に基づいて、React (TypeScript) を使用した「月次実績入力画面」のコンポーネントコードを作成・改善してください。

# 前提条件・技術スタック
- フレームワーク: React + Vite
- スタイリング: Tailwind CSS
- UIライブラリ: shadcn/ui (Radix UIベース)
- アニメーション: framer-motion (Reorder コンポーネントでの並び替え用)
- アイコン: lucide-react
- API通信: `@workspace/api-client-react` (OrvalによってOpenAPIから生成された React Query フック群)
- 画面の役割: 各社員の出勤日数・残業時間などの「月次実績」を一覧で入力し、さらに社員個別の「基本給・手当・控除」をサイドパネルで入力・計算プレビューする画面（URL: /monthly-input）。

# 使用するAPIフックとエンドポイント
【データ取得用】
- `useListEmployees({ active: true })`: 有効な社員一覧
- `useGetCompany()`: 会社マスタ（日給単価、保険料率などの取得用）
- `useListMonthlyRecords({ year, month })`: 対象月の実績一覧
- `useListAllowanceDefinitions({ activeOnly: true })`: 手当マスタ一覧
- `useListDeductionDefinitions({ activeOnly: true })`: 控除マスタ一覧
- `useGetEmployeeAllowances(employeeId)`: 社員ごとの手当設定
- `useGetEmployeeDeductions(employeeId)`: 社員ごとの控除設定

【データ更新用】
- `useCreateMonthlyRecord` / `useUpdateMonthlyRecord`: 月次実績の保存
- `useUpdateEmployeeAllowances` / `useUpdateEmployeeDeductions`: 社員別手当・控除の保存
- `useUpdateEmployee`: 社員の基本給の更新

※特別なAPI: 勤怠実績の自動取り込みは `fetch('/api/attendance/monthly-summary?year=...&month=...')` を使用します。

# UI/UX 要件
この画面は「メインの一括入力テーブル」と「個別の手当入力サイドパネル」の2つの主要要素で構成されます。

1. ページヘッダー・アクションバー:
   - `<AppLayout>` で全体をラップする。
   - 「年・月」を選択するセレクトボックス（デフォルトは現在の年月）。
   - 「勤怠データを取り込む」ボタン（クリックでfetchを呼び出し、結果を入力状態にマージする）。
   - 「一括保存」ボタン（テーブルで編集された全社員の月次実績を保存）。

2. メイン入力テーブル (エクセルライクなUI):
   - 社員ごとに1行となるテーブル。
   - カラム構成: 氏名、平日出勤(日)、土曜出勤(日)、日曜(h)、欠勤日数、残業(h)、深夜(h)、走行距離(km)、配送件数、備考、操作。
   - 各セルの数値は `<Input type="number" />` 等を用いてインラインで直接編集できるようにし、Reactのローカルステート（`edits` オブジェクト等）で管理する。
   - 入力値が不正（例：マイナス値など）な場合は赤枠で警告する簡易バリデーションを実装。
   - 操作カラムには「手当・控除設定」を開くボタンを配置。

3. 手当・控除入力サイドパネル (shadcn/ui の Sheet を使用):
   - テーブルの操作ボタンを押すと、右側から `<Sheet>` パネルがスライドインする。
   - パネル上部 (支給セクション): 
     - 「基本給」入力欄。日給制（`salaryType === 'daily'`）の社員の場合は、会社マスタの単価と入力された日数から自動計算して読み取り専用で表示。
     - 「各種手当」入力行。`<Select>` で手当種類を選び、`<Input>` で金額を入力。`framer-motion` の `Reorder.Group` を使い、ドラッグ＆ドロップで表示順を入れ替えられるようにする。
   - パネル中部 (社会保険料プレビュー):
     - 基本給＋課税手当の合計から、会社マスタの料率（健康保険・厚生年金・雇用保険）に基づき、概算の社会保険料を計算し、読み取り専用で表示。
   - パネル下部 (控除・差引支給額セクション):
     - 「各種控除」入力行（手当と同様に追加・削除・並べ替え可能）。
     - 最後に、総支給額から保険料と控除額を引いた「差引支給額（手取り概算）」を大きくハイライト表示する。
   - パネル最下部の「手当を保存」ボタンで、その社員の基本給・手当・控除を非同期で保存し、関連するQueryキャッシュを無効化する。

4. 通知・フィードバック:
   - `import { useToast } from "@/hooks/use-toast";` を使用し、取り込み、保存完了、エラー時にそれぞれトーストを表示。
